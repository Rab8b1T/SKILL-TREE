# Virtual Contest Platform - Quick Start Guide

## Getting Started

### Step 1: Access the Platform
1. Open the SKILL TREE main page
2. Click the **"Contest"** button (pink gradient button in the header)
3. You'll be redirected to the Virtual Contest platform

### Step 2: Load Your Profile
1. Enter your Codeforces handle (default is "rab8bit")
2. Click **"Load Profile"** or press Enter
3. Wait for the system to fetch your data:
   - User information
   - Solved problems list
   - Available problemset
4. Your stats will be displayed (handle, rating, problems solved)

### Step 3: Select Contest Type

#### Preset Divisions
- **DIV 4**: Perfect for beginners (800-1500 rating)
- **DIV 3**: Lower to mid-level (800-1700 rating)
- **DIV 2**: Intermediate level (1000-2200 rating)
- **DIV 1**: Expert level (1500-2800 rating)

#### Custom Contest
1. Click **"Custom"** card
2. Configure:
   - Number of problems (3-10)
   - Duration (60-180 minutes)
   - Min/Max rating range
   - Select topics/tags (optional)
3. Click **"Generate Contest"**

### Step 4: During the Contest

#### Solving Problems
1. Click on any problem name to open it in Codeforces
2. Solve the problem
3. Submit your solution on Codeforces
4. Return to the contest page

#### Tracking Progress
- **Auto-refresh**: Every 30 seconds
- **Manual refresh**: Click "Refresh Submissions" button
- **Keyboard shortcut**: Press 'R' key

#### Understanding the Interface
- **Timer**: Shows remaining time (changes color as time decreases)
- **Problems Table**: Shows all problems with current score
- **Status Badges**:
  - Gray: Not attempted
  - Yellow: Attempted (wrong submissions)
  - Green: Solved (accepted)
  - Red: Failed attempt

#### Score System
- Base score starts at 500 + (rating/10)
- Decreases by 0.2% per minute
- Minimum score is 30% of base
- Wrong submissions add 5-minute penalties

### Step 5: Contest End

#### Automatic End
- Contest ends when timer reaches 0:00:00

#### Manual End
- Click **"End Contest"** button
- Or press **Escape** key and confirm

### Step 6: View Results

#### Summary Section
Shows:
- Problems solved (X/Total)
- Total score earned
- Time taken
- Penalty minutes

#### Problem-wise Performance
Table showing:
- Each problem status
- Score earned
- Time to solve
- Number of attempts

#### Actions
- **View Past Contests**: See your contest history
- **Start New Contest**: Begin another contest

### Step 7: Explore Analytics

#### Past Contests History
- Filter by division type
- View all contest records
- See detailed statistics

#### Performance Graphs
- Switch between divisions (Overall/DIV4/DIV3/DIV2/DIV1)
- View score trends
- Track solve rate percentage
- See average statistics:
  - Average score
  - Average problems solved
  - Best score
  - Total contests participated

## Tips & Tricks

### Maximizing Score
1. Solve easier problems first (A, B, C)
2. Minimize time spent on each problem
3. Avoid wrong submissions (5 min penalty each)
4. Monitor the timer and score decay

### Contest Strategy
1. **Quick scan**: Read all problems first (2-3 minutes)
2. **Identify easy ones**: Start with problems you're confident about
3. **Time management**: Don't spend too long on one problem
4. **Penalty awareness**: Test your code thoroughly before submitting

### Using Custom Contests
1. **Rating range**: Start with narrower ranges for consistent difficulty
2. **Topic selection**: Focus on topics you want to practice
3. **Problem count**: Fewer problems (4-6) for focused practice
4. **Duration**: Adjust based on your available time

### Tracking Progress
1. **Regular contests**: Try to do 2-3 contests per week
2. **Division progression**: Start with DIV4, move up as you improve
3. **Analytics**: Review your performance graphs regularly
4. **Patterns**: Identify which problem types you struggle with

## Keyboard Shortcuts

- **Enter**: Submit handle / Start actions
- **R**: Refresh submissions during contest
- **Escape**: End contest (with confirmation)

## Understanding Status

### Problem Status
- **Pending**: You haven't attempted this problem
- **Attempted**: You have wrong submissions
- **Solved**: You got Accepted verdict
- **Failed**: Wrong answer detected

### Submission Status in Panel
- **Green border**: Accepted solution
- **Red border**: Wrong answer

## Common Issues

### "User not found"
- Check spelling of handle
- Verify handle exists on Codeforces
- Try again after a few seconds

### "Failed to fetch submissions"
- Check internet connection
- Codeforces API might be down
- Try refreshing manually

### Problems not updating
- Click "Refresh Submissions" button
- Wait 30 seconds for auto-refresh
- Ensure submissions are on Codeforces (not locally)

### No unsolved problems found
- You've solved too many problems in this rating range
- Try a different division
- Use custom contest with higher ratings
